# A500 ETF 量化工具 需求说明

## 问题

手动跟踪 A500 ETF（563360）的 K 线走势费时费力，缺乏系统化的技术指标辅助判断波段买卖点。需要一个本地工具来获取数据、计算常用技术指标、生成买卖信号并可视化，帮助做间隔一天到几天的中线波段决策。

## 交易哲学

详见 PROJECT_CONTEXT.md。核心原则：
- **弱者体系**——承认是弱者，只在自己有优势的领域出手
- **极端交易**——只在多指标共振指向极端位置时出手，平时观望
- **简单有效**——技术指标共振，不碰 ML/DL
- **回撤优先**——止损硬线 > 盈利目标
- **低频高质**——信号稀缺是优势

## 用户与场景

达达（个人投资者）在桌面电脑上运行命令行工具，查看 563360 的最新 K 线数据、技术指标和波段信号，辅助人工判断交易时机。

## MVP 范围

- 从 akshare 获取 563360 的日 K 线数据
- 本地缓存数据，避免重复请求
- 使用 pandas-ta 计算技术指标：MA5/MA10/MA20/MA60、MACD、RSI(14)、KDJ、布林带(20,2)
- **极端条件信号**：仅在多指标共振指向极端位置时生成买卖信号，默认保持观望
- 信号间隔可调（`--interval` 参数），间隔期内不重复判断
- 用 mplfinance 绘制含指标和信号的 K 线图
- 支持自定义日期范围参数

## 信号规则（具体·回测确认版）

### 前置开关

- ADX(14) ≥ 28 → 趋势市，今日不操作
- ADX(14) < 28 → 震荡/弱趋势市，进入信号判断

### 买入条件（三取二）
- 价格 ≤ 布林下轨 × 1.02
- RSI(14) < 35
- KDJ 金叉（K 上穿 D）且 K < 50

### 卖出条件（三取二）
- 价格 ≥ 布林上轨 × 0.98
- RSI(14) > 65
- KDJ 死叉（K 下穿 D）且 K > 50

### 信号间隔规则
- 同向信号发出后，5 个交易日内不再重复触发同向信号
- 避免价格在布林带外连续多天导致信号扎堆

### 不满足任何条件 → 观望

## 分批进出规则

买入信号亮 → **先买一半**：
- 3 个交易日内价格继续下跌 2% 以上 → 补另一半
- 3 个交易日内价格直接涨 → 不再追加，持有半仓等卖

卖出信号亮 → **先卖一半**：
- 3 个交易日内价格继续上涨 2% 以上 → 补卖另一半
- 3 个交易日内价格直接跌 → 不再追加，留半仓等下一个信号

## 止损规则（内置在工具中，不自动化执行，但输出提示）

1. **硬止损**：买入后价格跌破成本价 5% → 无条件离场，不犹豫
2. **时间止损**：买入后 7 个交易日未达到预期涨幅 → 离场
3. **总账户回撤**：该波段账户总资产回撤达 15% → 停止交易休整 2 周
4. **止损和分批补仓的关系**：已有半仓触发止损即离场，不补仓；半仓止损不计入补仓条件

## 非目标

- 不实现自动化下单或实盘交易
- 不做高频交易或日内（T+0）交易
- 不接入券商 API
- 不做多 ETF 横向对比
- 不使用 ML/DL 模型

## 输入与输出

- 输入：ETF 代码（563360 已内置）、日期范围（可选）、间隔天数（可选）
- 输出：终端打印信号摘要、保存 K 线图到本地、可选导出数据到 CSV

## 验收标准

- [ ] 运行 `python main.py` 即可获取最新数据并打印信号摘要
- [ ] 支持 `--days 90` 等参数调整分析窗口
- [ ] 支持 `--interval 3` 等参数调整信号间隔
- [ ] K 线图包含 MA、布林带、MACD、RSI、成交量子图
- [ ] 信号摘要明确显示买入/卖出/观望及判断依据
- [ ] 信号只在极端条件触发（默认大部分时间输出"观望"）

## 风险与依赖

- akshare 接口可能因上游变动而失效，需预留备用数据源
- 所有信号仅为辅助参考，不构成投资建议
- 极端条件策略可能导致长时间无信号，需要达达接受"等待"本身是策略的一部分
